Stratax AI is an individually developed interview simulation and evaluation platform. Unlike typical tools focusing purely on code syntax or static Q&A banks, Stratax evaluates reasoning depth, communication clarity, and decision-making. It replicates the nuanced, session-level assessment of a real human interviewer.

What Makes Stratax Different

We focus on the human signals that actually matter in a real interview.

Session-level evaluation

We grade your entire session's consistency and depth, moving beyond simple per-question gamification. Your ability to maintain context over a 45-minute span is what gets measured.

Conversational follow-ups

Dynamic, AI-driven probing questions that act like a real human. If you mention a trade-off, the agent will pivot to ask you to justify it based on scale.

Explainable scoring

Scores are meaningless without "why". You get exact strengths, gaps, and red flags mapped to rubrics.

Communication signals

Granular tracking of speech pace and filler words using intelligent audio processing.

Claim-based probing

Questions strictly target your specific resume claims ensuring tailored rigor.

Unified scoring engine

One powerful analysis pipeline backing your practice runs and interviews for a consistent standard across all interactions.

Environment Variables

Variable Required Description
GROQ_API_KEY One of Groq/Gemini Groq API key for Llama/Mixtral
GEMINI_API_KEY One of Groq/Gemini Google Gemini API key
DATABASE_URL Recommended PostgreSQL connection string

REDIS_URL Optional Redis for caching & rate limiting
JWT_SECRET_KEY Production Persistent JWT signing key
COOKIE_SECRET Production Cookie encryption secret
Production Notes Always use PostgreSQL (not SQLite), set persistent JWT_SECRET_KEY and COOKIE_SECRET, and enable STRATAX_SECRETS_ENCRYPTION_KEY for encrypted API key storage.

Security & Privacy

Built with privacy-safe design principles from the ground up.

  • Privacy-safe identifiers: SHA256/HMAC-SHA256 hashed user IDs in telemetry
  • No raw audio storage: Audio files are session-only, never used for learning
  • Resume privacy: Raw uploaded files deleted after extraction
  • Encrypted API keys: User-stored provider keys encrypted with Fernet
  • JWT + OAuth: Stateless JWT authentication with Google OAuth integration
  • Rate limiting: Tier-based rate limiting with Redis-backed coordination
  • CORS: Configurable origin whitelist
  • Identity Guard: System prompt prevents AI impersonation
